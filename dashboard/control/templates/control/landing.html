Multi-Tenant Architecture

A look under the hood. Isolated, containerized bot runtimes managed by a unified control panel.

1. Control Panel

Define token connection details, select custom plugins, and trigger container build processes.

2. Docker Daemon

Spins up a lightweight, sandboxed container specific to your bot type. Mounts persistent SQLite/MongoDB storage adapters on deployment.

3. Bot Instances

Establishes secure, long-poll API connections directly with Telegram or GitHub. Ready to process events instantly.
